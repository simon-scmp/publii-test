This is a test site for me using Publii

Hopefully published on Github and served off Cloudflare Pages

It has been a tedious move having looked at most of the Jamstack pile and beyond. Grav was going to be my tool of choice, it works well and offers a lot, but it does need PHP, runs off the server and feels, perhaps not quite of it’s time any more? When Static seems to be the way to go for simple sites. I started with static, writing html then progressing through GoLive, Freeway, Dreamweaver. Feels like it’s come full circle but different.

For historical reasons Plone was in there as well I’ve been using Plone for years and whilst it has quirks it is stable, solid and works well.

The others which came close were Jekyll the Ruby static site generator tightly integrated with GitHub, it runs GH Pages I’ve used this before and it works well.

I’m just starting to learn the frustrations and quirks of using Publii’s desktop editor. It’s near impossible to actually get it to save and publish to the path on an external drive which I set, always defaulting to the internal user directory on my Mac. It’s not easy to swap in and out of the three types of editor, WYSIWG, Blocks and Markdown. I tend to prefer to write in iA Writer and copy/paste across the MD so for default in Publii I’ve set to the Markdown editor. Because Publii isn’t allowing me to save in the cloud moving between machines and carrying on working isn’t going to be easy. I rely on being able to start on one device then carry on with a different one and just pick up. I started in the WYSIWG but not sure. Most of the grunt normally is done in the excellent BBEdit which seems to have been on the Mac longer than the Mac. It’s the most dependable, might not be the trendiest but is still the best if you have a lot of all sorts of text based projects to handle. I’m not just talking programming and coding here there is more to text than code and caffeine.
